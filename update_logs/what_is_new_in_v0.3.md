# v1.0.1 — 地铁系统集成

## 新增功能

### 1. 地铁渲染
- 新增 `metro-layer` 与 `metro-stations-layer` SVG 图层
- 根据 `roads/metro/metro.js` 数据绘制地铁线路（彩色实线）、站点（圆形，换乘站白色、非换乘站线路色）、线路标识（彩色方块+白字编号）
- 入口标识为极小淡红点（`r=0.5`，`opacity=0.35`）

### 2. 模式切换
- 控制面板新增"🚇 公共"模式按钮
- 仅在该模式下显示地铁图层；查看/导航模式自动隐藏
- 公共模式下导航控件可见，路径算法包含地铁

### 3. 地铁导航
- **图结构**：地铁站点、出入口、线路序列边加入 `allNodes` / `allRoads` / `graph`
- **地铁优先级**：`-2: 6`（快速路 3 的 2 倍）
- **出入口连接**：每个入口自动匹配最近道路节点（距离 ≤50）
- **登车惩罚**：`15 + (入口↔站台距离 / 2)` 最窄路线单位
- **换乘惩罚**：`10 + 两站中心直线距离` 最窄路线单位
- **铁路-地铁换乘**：站间距，中等道路单位
- **模式过滤**：非公共模式寻路时跳过所有地铁相关边

### 4. 路线着色
- 地铁行驶段按线路颜色绘制
- 换乘/出入口/道路段保持默认红色
- 使用 `.route-segment` 类，样式与主路线一致

### 5. 车站登车分级
- 铁路站数据新增可选字段 `boardCost`，覆盖默认 `CONFIG.railwayPenalty`（50）
- A东站设为 `boardCost: 120`，登车成本更高，使 A 站优先被使用

### 6. 界面优化
- 缩放至隐藏城市名称时（`zoom > 1.8`），起点/终点标记半径、`stroke-width` 同步减半，脉冲动画同步缩小
- 路线信息文本：地铁部分显示`（需乘地铁）` / `（需乘地铁，需换乘N次）`
- 出入口坐标修正：C1_1、C1_4、A4_1、A4_2 共 4 处入口坐标校准

## 核心算法/思想

**分层图导引（Layered Graph Routing）**：
- 道路网络为`基础层`，铁路为`快速层`，地铁为`高优先级层`
- 各层通过"换乘边"（Transfer Edge）连接，换乘边附加独立惩罚权重来模拟换乘成本（登车、步行、等车时间等）
- 路径搜索时按模式（公共/非公共）动态开启/关闭图层边，一阶段 Dijkstra 即可找到最优组合路径
- 权重公式：`weight = distance / priority`，优先级越高路径越被偏好，惩罚以"等效道路距离"归一化到统一标度
